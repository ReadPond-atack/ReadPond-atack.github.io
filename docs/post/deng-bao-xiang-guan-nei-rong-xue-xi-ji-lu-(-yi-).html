等保相关内容学习记录(一)

一、什么是等保

  "等保2.0"，是指以 2019 年发布的新一代网络安全等级保护标准为标志的等级保护制度，以《中华人民共和国网络安全法》为上位法，第二十一条明确"国家实行网络安全等级保护制度"，使等保成为网络运营者的法律义务。以 2019 年发布的GB/T 22239‑2019《信息安全技术 网络安全等级保护基本要求》为核心代表，替代2008版的GB/T 22239‑2008，因此业内习惯称为"等保2.0"。
  对比"等保1.0"，等保2.0扩展到"云、移、物、工、大"，包括：基础信息网络 / 平台、云计算平台 / 系统、大数据平台 / 应用、物联网系统、工业控制系统、采用移动互联技术的系统等。在GB/T 22239‑2019基本要求中采用"安全通用要求 + 安全扩展要求"结构，安全通用要求有物理、通信网络、区域边界、计算环境、管理中心等层面。安全扩展要求有云计算、移动互联、物联网、工业控制系统和大数据等扩展要求。
  "等保2.0"的防护理念从"被动防御"走向"主动防御、整体防控"。提出"一个中心、三重防御"的框架：其中一个中心为安全管理中心，三重防御是指安全通信网络、安全区域边界、安全计算环境。
  "等保2.0"对比之前定级流程更加严格，二级以上系统需要专家评审+主管部门审核，关键信息基础设施原则上定级为第三级及以上。

二、等保 2.0核心标准

  GB/T 22239‑2019《信息安全技术 网络安全等级保护基本要求》大致作用为核心标准，规定一至四级的安全通用要求和云计算、移动互联、物联网、工业控制、大数据等扩展要求，是“建设整改和测评的直接依据”。
  GB/T 28448‑2019《信息安全技术 网络安全等级保护测评要求》大致作用规定各等级的测评要求、测评方法和单元测评，用于指导等级测评机构开展测评工作。
  GB/T 25070‑2019《信息安全技术 网络安全等级保护安全设计技术要求》从“一个中心、三重防护”出发，规定不同等级系统的安全设计技术要求，用于指导安全方案设计。

三、配套工作与实施类标准、基础与关联标准

  GB/T 22240‑2020《信息安全技术 网络安全等级保护定级指南》指导如何确定定级对象、划分安全保护等级。
  GB/T 25058‑2019《信息安全技术 网络安全等级保护实施指南》给出等级保护工作的总体实施流程和方法，适合作为“总说明书”。
  GB/T 28449‑2018《信息安全技术 网络安全等级保护测评过程指南》规范测评准备、方案编制、现场测评、报告编制等过程。
  GB/T 36958‑2018《信息安全技术 网络安全等级保护安全管理中心技术要求》等对安全管理中心等提出技术要求，是“一个中心”的落地支撑。
  GB 17859‑1999《计算机信息系统安全保护等级划分准则》最早的等级划分基础标准，定义了五个安全等级。
  GB/T 20984‑2022《信息安全技术 信息安全风险评估方法》最新版风险评估国家标准，用于指导信息安全风险评估工作，是等保之外的重要方法类标准。
  其他如 GB/T 35273‑2020《个人信息安全规范》、GM/T 0054‑2018《信息系统密码应用基本要求》等，常作为等保建设中的补充安全要求。

四、等保 2.0 与风险评估的关系

  本质上，等级保护是国家网络安全基本制度，是一套完整的制度体系（定级、备案、建设整改、测评、监督检查）。而风险评估则是信息安全风险管理中的技术手段，是“识别风险、分析风险、评价风险”的过程。
  从目标层面等保2.0解决“该达到什么安全水平”“是否合规”的问题，强调按等级建设和监管，风险评估解决的是“当前风险有多大”“哪些风险需要优先处理”的问题，支撑风险决策和安全需求分析。
  从视角层面，等保更偏向于合规 + 建设，而风险评估更偏向于风险+管理。

五、GB/T 22239-2019

1. 整体框架

  前言、引言，说明标准修订背景（适配新法与新技术）、与旧版的核心差异、关联标准体系（如定级指南 GB/T22240、测评要求 GB/T28448 等），并明确黑体字为高等级新增 / 增强要求。
  1-5章为规定标准范围（指导非涉密对象 1-4 级安全建设）、规范性引用文件，界定网络安全、云计算、物联网、工业控制系统等核心术语与缩略语；核心阐述网络安全等级保护概述，明确等级保护对象（基础信息网络、云平台、物联网、工控系统等）、1-5 级安全保护能力要求，以及安全通用要求 + 安全扩展要求的核心分类体系。
  6-10章标准核心内容，按 1-5 级依次规定安全要求，其中第五级因特殊管理模式仅作提及，无具体条款；每一级要求均分为安全通用要求（覆盖安全物理环境、安全通信网络、安全区域边界、安全计算环境、安全管理中心 5 大技术维度，及安全管理制度、机构、人员、建设管理、运维管理 5 大管理维度）和安全扩展要求（针对云计算、移动互联、物联网、工业控制系统 4 大新技术场景提出个性化要求）。
  附录部分则明确安全要求的选择使用方法、等级保护对象整体安全保护能力要求、安全框架与关键技术使用要求；资料性附录详细说明云计算、移动互联、物联网、工业控制系统、大数据的应用场景，及各场景下安全要求的适配方法。

2. 各个级别的核心差异

  二级要求的核心是“基础可控、流程规范”，满足中小型组织、一般重要信息系统的安全防护需求，以 “配置 + 制度” 为主，无需高强度密码技术、专职安全团队、集中管控平台；三级要求的核心是“纵深防御、集中管控、专业保障”，满足大中型组织、重要信息系统（如政府部门、金融机构核心系统）的安全防护需求，新增密码技术全面应用、硬件冗余、集中管控、专职安全团队、第三方监督等要求，防护能力从 “基础可控” 提升至 “主动防御、集中管理”。

3. 技术要求强度差异

  技术要求围绕 “物理环境、通信网络、区域边界、计算环境、管理中心”5 大维度，呈现从“基础配置”到 “冗余加固、密码技术、可信验证”，从 “单一防护” 到 “多层防护、集中监测”的递进：1-2 级以基础防护为主，如物理环境仅需基本的门禁、防火、供电；计算环境仅需基础身份鉴别、防恶意代码；3级新增密码技术应用（数据传输 / 存储的机密性 / 完整性）、硬件冗余（通信线路、电力）、可信验证（动态检测）、集中管控等要求，如身份鉴别要求双因子认证、访问控制粒度到用户 / 文件级；4级进一步强化高强度密码技术（硬件密码模块）、双重物理防护（重要区域二道门禁）、异地灾备中心（业务实时切换）、全环节动态可信验证、强制访问控制等要求，技术防护的强度、覆盖度、智能化程度大幅提升。

4. 管理要求精细化程度差异

  管理要求围绕“制度、机构、人员、建设、运维”5 大维度，呈现从 “基础制度 / 岗位” 到 “专职机构 / 人员、全流程审批、定期测评、集中审计”的递进：1级仅需建立基础管理制度、设立系统管理员岗位；2级要求制定完整安全策略、设立安全管理职能部门、明确岗位分离、定期备份与应急演练；3级要求成立网络安全委员会 / 领导小组、配备专职安全管理员（不可兼任）、全流程审批（系统变更、物理访问）、定期等级测评、第三方工程监理，管理流程更精细化、规范化；4级进一步要求关键岗位多人共管、内部选拔关键岗位人员、关键区域禁止外部人员访问、跨单位联合应急、重要产品专项测试，管理的严格性、专业性、协同性大幅提升。

5. 扩展要求适配性差异

  针对云计算、移动互联、物联网、工控系统的扩展要求，随等级提升呈现从 “基础隔离 / 管控” 到 “高强度认证 / 审计、主动检测 / 阻断、全生命周期管理”** 的递进：2级扩展要求仅需基础隔离（如云平台虚拟网络隔离、工控系统与企业网技术隔离）、基础管控（如移动应用可靠签名、物联网感知节点授权接入）；3级新增主动检测 / 阻断（如移动互联阻断非授权终端、工控系统检测非法无线设备）、高强度认证（如移动互联采用密码模块认证）、全生命周期管理（如物联网感知节点全流程管理）；4级进一步强化资源独立（如云平台为四级业务划分独立资源池）、单向强隔离（如工控系统与企业网专用产品单向隔离）、专用管控（如移动终端仅处理指定业务），适配新技术场景的安全防护要求更严苛。

六、总结

网络安全等级保护是依据保护对象在国家安全、经济建设等中的重要程度及受损后的危害程度划分等级，为非涉密对象制定对应等级的安全通用与扩展要求，指导其安全建设和监督管理的网络安全防护制度。等级保护1-4级要求从安全保护能力、技术防护强度、管理精细化程度、扩展要求适配性上呈现出从基础防护到纵深防御、从人工零散管理到专业集中管控、从单一基础管控到智能高强度防护的逐级递进特征。网络安全等级保护工作中需结合风险评估结果补充和调整安全措施，风险评估是等级保护对象确定安全保护等级、选择适配安全要求的重要依据，二者相辅相成共同保障保护对象的网络安全。
